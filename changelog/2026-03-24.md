### Added
- **AWSManagedRulesSQLiRuleSet** (priority 2): SQL injection detection managed rule group, inserted between CommonRuleSet and KnownBadInputsRuleSet.
- **Path-specific rate limiting** (priority 5): 200 requests/5-minute window per IP, scoped to `POST /receipts/upload` via `AndStatement` combining `ByteMatchStatement` on method (`POST`) and URI path (`/receipts/upload`, `EXACTLY` positional constraint). Prevents abuse of the receipt upload presigned-URL endpoint without affecting general API traffic.
- **WAF access logging**: Dedicated CloudWatch LogGroup (`/aws/wafv2/{prefix}-waf-logs`) with 90-day retention. Implemented as a separate `AWS::WAFv2::LoggingConfiguration` resource (CDK 2.241+ requirement — not a property on `CfnWebACL`). Authorization header redacted from logs to prevent Cognito token leakage.
- **WAF LogGroup**: 90-day retention, environment-aware removal policy (RETAIN in prod, DESTROY otherwise).

### Changed
- **Global rate limit**: Increased from 2,000 to 5,000 requests/5-minute window per IP to reduce false positives at scale.
- **Rule priorities**: Reordered to accommodate new SQLi rule — KnownBadInputs shifted to priority 3, global rate limit to 4, receipt upload rate limit to 5.
- **WebACL description**: Added descriptive text (`"CloudFront WAF: managed rules + rate limiting"`).

### Notes

## [SRE-P2] — Circuit Breaker for Downstream AWS Services

### Added
- **Circuit breaker utility** (`backend/src/utils/circuitBreaker.ts`): State machine with three states — `CLOSED` (normal), `OPEN` (rejecting), `HALF_OPEN` (probing recovery).
  - Configurable `failureThreshold` (default: 5 consecutive failures) and `cooldownMs` (default: 30s).
  - Injectable `now()` clock for deterministic testing without `vi.useFakeTimers`.
  - HALF_OPEN allows exactly 1 probe request; additional calls are rejected with "probe limit" error.
  - `CircuitOpenError` includes breaker name and remaining cooldown in message.
  - `_getState()` / `_setState()` test helpers with shallow-copy immutability.

- **Textract circuit breaker** in `sns-webhook` handler:
  - Wraps `GetExpenseAnalysis` call.
  - On `CircuitOpenError`: writes receipt to `pending_manual_review` status via `writeScanPendingManualReview()`, skipping Bedrock categorization.
  - Emits `TextractCircuitOpen` CloudWatch metric.

- **Bedrock circuit breaker** in `sns-webhook` handler:
  - Wraps `InvokeModel` (Claude categorization) call.
  - On `CircuitOpenError`: falls back to keyword-based categorization (`merchantName` heuristic).
  - Emits `BedrockCircuitOpen` CloudWatch metric.

- **Unit tests** (`backend/__tests__/unit/circuitBreaker.unit.test.ts`): 25 tests covering all state transitions, probe limiting, error propagation, lazy `getState()` evaluation, and immutability of `_getState()` snapshots.

### Fixed
- **Circuit breaker test type mismatch** (line 182): `resolveProbe` typed as `() => void` but assigned from `Promise<string>` resolve callback — widened to `(value: string) => void`.
- **Missing `vi` import** in circuit breaker tests: Added `vi` to vitest import for `vi.fn()` usage in OPEN and HALF_OPEN probe tests.

### Degradation Strategy
| Service | Circuit Open Behavior | User Impact |
|---------|----------------------|-------------|
| Textract | Receipt → `pending_manual_review` status | Manual processing queue |
| Bedrock | Keyword-based fallback categorization | Reduced categorization accuracy |

### Test Results
- 184 unit tests passing (↑25 from circuit breaker suite)
- 54 integration tests passing

### Notes
- Circuit breakers are per-Lambda-invocation (not persisted across invocations) — AWS Lambda's stateless model means each cold/warm start gets a fresh breaker. Persistent circuit state would require DynamoDB or ElastiCache backing.
- `pending_manual_review` status requires a frontend workflow to surface and resolve stuck receipts (future work).

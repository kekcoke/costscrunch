- **Frontend** (`frontend/src/pages/settings.tsx`):

## Expense Export Lambda — Dedicated Handler with S3 Presigned URLs
- **Gap**: Export functionality was embedded in expenses Lambda; large datasets (>1000 items) needed S3 presigned URL support.
- **New Lambda** (`backend/src/lambdas/expense-export/index.ts`):
    - `GET /expense-export` — queries DynamoDB with pagination, generates CSV/JSON.
    - Large datasets (≥1000 items): uploads to S3 under `exports/{userId}/{ulid}.csv`, returns presigned URL (30-min expiry).
    - Small datasets: returns inline `{ data, format, count }` JSON.
    - Filters: `status`, `category`, `from`, `to`, `groupId`, `format` (csv/json).
    - Uses `csv-stringify/sync`, PowerTools Logger/Metrics.
- **Backend Route** (`backend/src/server.ts`):
    - Added import for `expenseExportHandler`.
    - Added `GET /expense-export` route.
- **Frontend** (`frontend/src/services/api.ts`):
    - `expensesApi.export()` now calls `/expense-export` endpoint (previously `/expenses/export`).
    - Handles both presigned URL download and inline CSV/JSON browser download.
- **Tests**:
    - `backend/__tests__/unit/expense-export.unit.test.ts` — 8 unit tests (CSV/JSON export, S3 upload threshold, filters, error cases).
    - `backend/__tests__/integration/expense-export.integration.test.ts` — 6 integration tests via LocalStack (full round-trip, group-scoped export).

## Infrastructure — CfnBucket Fixes & TypeScript Errors
- **Gap**: CDK's high-level `s3.Bucket` doesn't support sub-day S3 lifecycle expiration. Quarantine bucket needed 5-hour expiration to minimize storage costs.
- **CDK Stack** (`infrastructure/stacks/CostsCrunchStack.ts`):
    - Switched `QuarantineBucket` from `s3.Bucket` (L2) to `s3.CfnBucket` (L1) to support `expirationInSeconds: 18000`.
    - L2 `grantPut()` not available on `CfnBucket` — replaced with explicit IAM policy: `s3:PutObject` on `quarantineBucket.attrArn/*`.
    - Fixed TypeScript errors:
        - `serverSideEncryptionRule` and `expirationInSeconds` not in CDK generated types → cast props `as any` (valid S3 API properties).
        - `environment` type mismatch on `sharedLambdaProps` → cast `as { [key: string]: string }`.
        - `quarantineBucket.bucketArn` → `quarantineBucket.attrArn` (CfnBucket doesn't have `bucketArn`).
- **Encryption Tests** (`infrastructure/__tests__/EncryptionAspect.test.ts`):
    - `Duration.hours(5)` error: CDK's `expiration` only accepts whole days → fixed by switching to CfnBucket with `expirationInSeconds`.
    - `grantPut` not a function: replaced with IAM policy statement.
    - All 3 tests now pass.

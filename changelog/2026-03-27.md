## 🚀 Fix
### Stale Container Management 
- **Root Cause**: The error was caused by trying to execute docker ps and `docker rm inside the costscrunch-localstack` container during the bootstrap phase. Since the LocalStack container does not have the Docker CLI installed or access to the host's Docker socket, these commands failed.
- **Changes (1)**: 1. Backend Script Cleanup: Removed the invalid Docker commands from infrastructure/localstack/opt2/bootstrap.sh. 
- **Changes (2)**: 2. Host Orchestration Update: Moved the cleanup logic to setup/localstack.sh. This script runs on your host machine, where it has the necessary permissions and tools to manage the Docker environment.
- **Changes (3)**: 3. Idempotent Cleanup: The script now performs a silent check for any containers matching the costscrunch-localstack-lambda pattern and force-removes them before starting the new LocalStack session. The bootstrap process is now robust and will correctly clear stale resources without throwing execution errors.

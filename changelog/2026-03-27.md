# Changelog — 2026-03-27
## Custom Authentication & Cognito Integration

Replaced standard AWS Amplify components with a high-control, vertically integrated authentication system. This migration enables custom branding, enhanced security logic (MFA/Refresh), and full local simulation.

---

### Phase 1: Infrastructure & CDK
- **Auth Lambda**: Defined `AuthLambda` and registered public routes in `CostsCrunchStack`.
- **Cyclic Dependency Fix**: Decoupled UserPool IDs from global environment variables to resolve synthesis errors.
- **Resource Protection**: Enforced `Retain` removal policies and MFA configurations based on stage.
- **Assertions**: Added 10 granular template tests for UserPool, Groups, and Client settings.

### Phase 2: Automation & Data Integrity
- **Post-Confirmation Trigger**: Implemented `auth-trigger` Lambda.
- **Profile Automation**: Automatically creates `USER#PROFILE` records in DynamoDB upon email confirmation.
- **Idempotency**: Added `ConditionalCheckFailedException` handling to prevent duplicate profile writes.

### Phase 3: Backend Service Layer
- **Cognito SDK Wrapper**: Created `authService.ts` using AWS SDK v3.
- **Feature Set**: Implemented `signUp`, `confirmSignUp`, `signInUser` (with challenge detection), `refreshAuth`, `forgotPassword`, and `confirmMfa`.
- **Error Mapping**: Standardized Cognito exceptions into actionable `AuthError` responses.

### Phase 4: Emulator Integration & Testing
- **cognito-local**: Integrated the Cognito emulator into the local triad (LocalStack + SAM + Emulator).
- **Provisioning**: Created `scripts/provision-local-cognito.js` for automated local resource setup.
- **E2E Integration Tests**: Verified full Register → Confirm → Login flow against the live emulator.

### Phase 5: Middleware & Security
- **JWT Utilities**: Created lightweight parser for extracting sub, email, and groups from tokens.
- **withAuth HOF**: Implemented higher-order function to protect routes and inject identity into Lambda context.
- **Validation**: 19 tests verifying claim extraction and token expiry logic.

### Phase 6: Frontend Migration
- **Auth API Helper**: Created `auth-api.ts` to replace `aws-amplify` dependencies.
- **UI Refactoring**: 
  - `LoginPage`: Added MFA challenge detection and context passing.
  - `RegisterPage`: Integrated backend confirmation flow.
  - `MFAPage`: Migrated to custom verification logic with countdown timers.
  - `PasswordResetPage`: Implemented dual-step reset flow.
- **Bundle Optimization**: Uninstalled `aws-amplify` library.

---
**Status**: 🌸 All 82 auth tests passing. 315 total backend tests verified.

# 2026-03-27 (Categorization, Sorting & Receipt Stability)

## 🚀 Features & Enhancements

### 🗃️ Advanced Expense Management
- **Categorical Filtering**: Added dedicated category filter dropdown to the All Expenses page, supporting server-side filtering for `Groceries`, `Travel`, `Software`, `Meals`, `Office`, `Equipment`, and `Other`.
- **Flexible Sorting**: Implemented sort order controls allowing users to arrange expenses by `Date (Newest/Oldest)` or `Amount (Highest/Lowest)`.
- **Strict Schema Enforcement**: Refactored backend and frontend to use a unified `CategoryName` type, ensuring data consistency across the entire stack.

### 📎 Receipt Pipeline Reliability
- **Persistence Fix**: Resolved a critical bug in the `sns-webhook` Lambda where `receiptKey` was not being persisted to DynamoDB after processing, which caused 404 errors during download.
- **Dynamic Local Seeding**: Updated LocalStack `setup.sh` to dynamically upload sample PDFs for every unique receipt path in `seed.csv`, ensuring immediate "View Receipt" availability for all test users.
- **Auth Standardization**: Standardized the `receipts` Lambda to use the central `getAuth` utility, improving security and consistency in user identity resolution.

## 🛠 Stability & Bug Fixes
- **Async Test Resilience**: Refactored frontend unit tests to use `await waitFor` and proper `useEffect` triggers, resolving race conditions during initial data load.
- **Mock Data Alignment**: Migrated legacy `Misc`, `Dining`, and `Utilities` categories in seed data and mocks to the official system categories (`Meals`, `Other`).
- **Lambda Export Refactor**: Exported `rawHandler` from backend Lambdas to allow unit tests to bypass middleware for more precise logic verification.

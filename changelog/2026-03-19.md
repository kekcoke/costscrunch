# Changelog — 2026-03-19

## Feature Branch: `feature-centralize-cors`

### Infrastructure (CDK)
- **Centralized CORS SSOT**: Defined a single source of truth for CORS configuration in `CostsCrunchStack.ts` (Origins, Methods, Headers).
- **CloudFront Response Headers Policy**: Implemented `corsHeadersPolicy` at the CDN layer to ensure all `/api/*` responses (including 4XX/5XX from API Gateway) carry valid CORS headers.
- **PATCH Method Support**: Explicitly added `PATCH` to the allowed methods in both API Gateway and CloudFront configurations.

### Backend (Lambda & Local Server)
- **Lambda Cleanup**: Removed redundant manual `CORS_HEADERS` and injection logic from `groups/index.ts`, reducing response payload and handler complexity.
- **Express CORS Alignment**: Refactored `backend/src/server.ts` to match the production CORS configuration for local development.
- **Adapter Header Stripping**: Updated `lambdaAdapter` to strip CORS headers returned by Lambda handlers, ensuring the Express middleware remains the sole authority in local mode.
- **App Export**: Exported the Express `app` instance to enable programmatic integration testing.

### Testing
- **New Integration Suite**: Created `backend/__tests__/integration/cors.integration.test.ts`.
  - Verifies `OPTIONS` preflight success (204 No Content) across all 13 API routes.
  - Validates consistent CORS header presence on actual resource requests (GET/POST/PATCH).
  - Confirms Lambda-level header overrides do not interfere with the centralized policy.

### Documentation
- **Technical Evaluation**: Authored `docs/cors-refactor-evaluation.md` detailing the transition from per-handler CORS to infrastructure-level enforcement and Hono feasibility.
- **README Updates**: Updated the "Local vs Production" parity table and added specific testing commands for CORS verification.


## Feature Branch: `cleanup/launch-blockers`

### Security
- **Runtime Guard (OWASP ASVS v4.0 V13.1)**: Added runtime check in `withMockAuth` that throws immediately if `MOCK_AUTH=true` is set outside `ENVIRONMENT=dev`. Handler invocation is terminated before any business logic runs.
- **CDK Synthesis Guard**: Added an `Aspects` scan in `CostsCrunchStack` that fails `cdk synth`/`cdk deploy` if any Lambda function carries a `MOCK_AUTH` environment variable in staging or prod. Error annotation includes the specific function path and OWASP reference.
- Guard is skipped entirely in dev — no overhead for local development.

### Testing
- **Runtime Guard Tests** (8 new): Validates throw for prod/staging/unexpected envs, allow for dev/false/unset, error message contains environment name and V13.1 reference.
- **CDK Synthesis Guard Tests** (8 new): Validates clean stacks pass, dev skips aspect, individual Lambda flagging with function path, multi-Lambda isolation, and mixed safe/unsafe Lambda scenarios.

### Planning
- **Product Roadmap**: Created `notes/roadmap.md` — 24 production-grade gaps across 8 categories (Security, Observability, Reliability, Scalability, Testing, DevOps, Frontend, Data Integrity) with priority/complexity ratings, estimated timelines, and copy-paste-ready AI prompts for iterative development of each gap.

## Feature Branch: `om-p1-structured-logging`

## Feature Branch: `om-p1-cloudwatch-sns-alerts`

### Infrastructure (Observability)
- **CloudWatch Alarms**: Implemented comprehensive system monitoring in `CostsCrunchStack.ts`:
    - **Error Rate**: Alarms if Lambda error rate > 5% over 5m (3 consecutive datapoints).
    - **Duration/Timeout**: Alarms if execution time exceeds 80% of the function's timeout.
    - **DynamoDB**: Immediate alerts on any `ThrottledRequests` in the main table.
    - **Pipeline Health**: Custom `MetricFilter` on `SnsWebhookLambda` logs for Textract/SNS failure patterns.
- **SNS Alerts**: Created an encrypted `AlarmsTopic` and wired all system alarms to it for centralized alerting.
- **Local Dev Parity**: Updated `setup.sh` to provision the alarms SNS topic in LocalStack, maintaining infrastructure consistency across environments.

### Testing
- **CDK Snapshot Suite**: Added `infrastructure/__tests__/CostsCrunchStack.test.ts`.
    - Verifies alarm thresholds, evaluation periods, and SNS topic wiring.
    - Implemented stable synthesis logic for unit tests by providing fallbacks for account/region Tokens.

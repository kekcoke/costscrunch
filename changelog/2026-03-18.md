# Changelog — 2026-03-18
## Feature Branch: `add-groups-add-missing-infra`

### Infrastructure & Orchestration
- **REST API v1 Migration**: Standardized local and SAM environments on API Gateway REST API (v1) for full feature parity with LocalStack free tier.
- **Hierarchical Routing**: Implemented `{proxy+}` resource mapping for dynamic path parameter handling in REST v1.
- **CORS Enforcer**: Created `infrastructure/localstack/opt2/enable-cors.sh` to programmatically configure preflight headers across the entire API tree.
- **Deployment Automation**: Added logic to force API deployments in LocalStack after resource modifications.
- **SAM Template Updates**: Aligned `template-arm.yaml` and `template-x64.yaml` with the REST v1 architecture and bundled Lambda handlers.

### Backend (Groups Service)
- **Soft-Delete Implementation**: Added `DELETE /groups/{id}` with ownership checks, pending expense blocks, and zero-balance enforcement.
- **Atomic Memberships**: Refactored group creation to use `TransactWriteItems`, ensuring the `GROUP_MEMBER` record is created alongside the `GROUP` profile.
- **Path Normalization**: Added `normalizeRoute` helper to bridge REST v1 actual paths with template-based routing logic.
- **Member Removal**: Implemented `DELETE /groups/{id}/members/{userId}` with financial integrity safeguards.
- **Data Compliance**: Fixed DynamoDB seed data types (e.g., `L` for members/splits) to prevent runtime serialization errors.

### Frontend
- **Groups Page**: Added real-time search filtering and sorting by `name`, `type`, `totalSpend`, and `monthSpend`.
- **Group Detail**: 
  - Integrated "Update Group" settings form with theme color selection.
  - Implemented member removal flow with confirmation modals.
  - Added "Danger Zone" for group soft-deletion.
  - Added search and sort filter.
- **Type Safety**: Updated `Group` and `GroupMember` interfaces to match the standardized backend model.

### Testing
- **New Test Projects**: 
  - `infrastructure/__tests__/opt2/bootstrap-v1.test.ts` (REST v1 validation).
  - `infrastructure/__tests__/data-seed.test.ts` (DDB compliance check).
- **Expanded Coverage**:
  - `backend/__tests__/integration/groups.integration.test.ts` (Full E2E lifecycle).
  - `frontend/__tests__/groups-page.test.tsx` (Filter/Sort logic).
  - `frontend/__tests__/group-detail.test.tsx` (Edit mode & Modal actions).

### CI/CD
- **Accelerated Deploys**: Implemented `SKIP_STAGING` environment flag in GitHub Actions to bypass staging and go straight to production.
- **CI Parity**: Updated `workflow.yml` to perform full LocalStack REST v1 provisioning (Compute Layer bootstrap) before running the integration suite.
- **Dynamic API Discovery**: Automated `API_ID` resolution in CI to ensure backend integration tests hit the correct REST v1 endpoint.

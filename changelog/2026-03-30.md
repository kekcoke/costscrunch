# Changelog — 2026-03-30

## [Authentication & Identity Lifecycle]

This session marks the completion of the `setup/landing-login-signup-mfa-passwordreset` feature branch, providing a robust, production-grade identity layer.

### Added
- **New Frontend Pages**: 
    - `landingPage.tsx`: Marketing home page with feature highlights and pricing tiers.
    - `loginPage.tsx`: Multi-provider sign-in (Email, Google/GitHub mocks) with integrated MFA challenge detection.
    - `registerPage.tsx`: Multi-step user enrollment (Registration -> Email Verification).
    - `mfaPage.tsx`: TOTP authenticator verification with real-time countdown sync.
    - `passwordResetPage.tsx`: Unified flow for requesting and confirming password resets.
- **Account Management**: 
    - Added soft-delete functionality via `DELETE /auth/account`, archiving profiles in DynamoDB and disabling Cognito access.
- **Resilient Infrastructure Testing**:
    - Implemented a "Self-Healing" integration test suite in `auth.integration.test.ts` that dynamically bootstraps and discovers Cognito resources in local emulators.

### Changed
- **Architectural Shift (Proxy Pattern)**: 
    - Migrated from client-side Amplify-to-Cognito calls to a **Secure Lambda Proxy** model.
    - Centralized all auth logic in `frontend/src/services/api.ts` (`authApi`).
- **Auth Service Layer**: 
    - Refactored `authService.ts` to use dynamic environment resolution, eliminating module initialization race conditions in Vitest.
- **Project Documentation**:
    - Updated `README.md` with new endpoints and structural notes.
    - Created `docs/auth-feature-changes.md` summarizing the technical implementation.

### Fixed
- **Module Lifecycle Issues**: Resolved circular dependency and environment injection failures in backend testing.
- **Git Hygiene**: Updated `.gitignore` to prevent committing transient emulator database state.

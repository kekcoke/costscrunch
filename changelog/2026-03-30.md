# Changelog — 2026-03-30

## [Authentication & Identity Lifecycle]
This session marks the completion of the `setup/landing-login-signup-mfa-passwordreset` feature branch, establishing a production-grade identity layer.

### Added
- **Core Auth Pages**:
  - `landingPage.tsx`: High-conversion marketing home page.
  - `loginPage.tsx`: Integrated login with MFA challenge detection.
  - `registerPage.tsx`: Multi-step enrollment with email verification.
  - `mfaPage.tsx`: TOTP verification with real-time countdown sync.
  - `passwordResetPage.tsx`: Unified recovery flow for forgotten passwords.
- **Backend Auth Lambda**:
  - Implementation of `forgot-password`, `confirm-password`, and `confirm-mfa` routes.
  - Added `DELETE /auth/account` for secure user-initiated soft-deletion.
- **Testing & Security**:
  - 31 passing unit and integration tests covering the full identity lifecycle.
  - Self-healing integration suite that bootstraps the Cognito emulator at runtime.

### Changed
- **Architectural Pivot**: Migrated from client-side Amplify-to-Cognito calls to a **Secure Lambda Proxy** model, enabling server-side hooks (e.g., automatic profile creation on first login).
- **Service Layer**: Centralized `authApi` in `frontend/src/services/api.ts` for unified endpoint management.
- **Refactoring**: Updated `authService.ts` for dynamic environment resolution to support varied local emulator IDs.

### Fixed
- **Git Hygiene**: Removed transient `.cognito/db/` and test-generated JSON files from version control.
- **Module Race Conditions**: Fixed Vitest initialization errors via dynamic imports and environment getters.

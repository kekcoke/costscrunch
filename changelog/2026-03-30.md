# Changelog — 2026-03-30

## [Authentication & Identity Lifecycle]
This session marks the completion of the `setup/landing-login-signup-mfa-passwordreset` feature branch, establishing a production-grade identity layer using a Secure Lambda Proxy pattern.

### Added
- **Authentication Pages**:
  - `landingPage.tsx`: Marketing home with feature highlights and pricing.
  - `loginPage.tsx`: Multi-provider sign-in with MFA challenge detection.
  - `registerPage.tsx`: User enrollment with email verification.
  - `mfaPage.tsx`: TOTP verification with real-time sync.
  - `passwordResetPage.tsx`: Unified password recovery flow.
- **Account Management**:
  - Soft-delete functionality via `DELETE /auth/account` (Archives DynamoDB profile + disables Cognito user).
- **Resilient Testing**:
  - Self-healing integration tests in `auth.integration.test.ts` with dynamic User Pool discovery.
  - Added 22+ unit tests in `auth-cognito.unit.test.ts`.

### Changed
- **Architectural Migration**:
  - Transitioned from direct Amplify-to-Cognito calls to a **Secure Lambda Proxy**.
  - Centralized auth logic in `frontend/src/services/api.ts` (`authApi`).
- **Backend Refactoring**:
  - `authService.ts`: Refactored for dynamic environment variable resolution, fixing module initialization race conditions in tests.
- **Documentation**:
  - Updated `README.md` with new endpoints and structure.
  - Created `docs/auth-feature-changes.md` summarizing the implementation.

### Fixed
- **Vitest Lifecycle**: Resolved issues with module hoisting and environment injection in backend tests.
- **Git Hygiene**: Cleaned up transient `.cognito/db/` emulator files from tracking.

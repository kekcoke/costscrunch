## File Upload Edge Cases & Quarantine Bucket
- **Gap**: Upload payload limits (Roadmap 4.3) — preventing oversized/unreadable files from DOSing the API.
- **CDK Stack** (`infrastructure/stacks/CostsCrunchStack.ts`):
    - Added `QuarantineBucket` with 5-hour expiration lifecycle rule.
    - Added `BUCKET_QUARANTINE_NAME` and `WEBSOCKET_ENDPOINT` to shared Lambda environment.
    - Added IAM permissions for quarantine bucket access.
- **Image Preprocess Lambda** (`backend/src/lambdas/image-preprocess/index.ts`):
    - Added `quarantineFile()` helper to move invalid files to quarantine bucket.
    - Added `notifyUserViaWebSocket()` for real-time user notification.
    - Added file validation: size check (10MB max), dimension check (10,000px max).
    - Added Sharp metadata validation to detect corrupt/unreadable files.
    - Failure reasons: `UNSUPPORTED_FORMAT`, `CORRUPT_FILE`, `DIMENSION_EXCEEDED`, `SIZE_EXCEEDED`, `SHARP_ERROR`.
- **SNS Webhook Lambda** (`backend/src/lambdas/sns-webhook/index.ts`):
    - Added multi-page detection (`isMultiPage` flag) when Textract returns multiple pages.
    - User is notified via WebSocket that only first page was processed.
    - Updated DynamoDB and EventBridge to include `isMultiPage` field.
- **WebSocket Notifier** (`backend/src/lambdas/web-socket-notifier/index.ts`):
    - Added `handleQuarantineEvent()` for QUARANTINE notifications.
    - Added `QuarantineEventDetail` interface.
    - Includes multi-page flag in `RECEIPT_SCAN_COMPLETED` payloads.
- **LocalStack** (`infrastructure/localstack/dev/setup.sh`):
    - Added quarantine bucket creation with 5-hour expiration.
    - Added SSM parameter for `quarantine-bucket`.
- **SAM Configuration** (`infrastructure/sam/env.json`):
    - Added `BUCKET_QUARANTINE_NAME` parameter.
- **Tests** (`infrastructure/__tests__/localstack/image-preprocess.test.ts`):
    - 20 tests covering quarantine reasons, file size limits, dimension limits, supported formats, key validation, and WebSocket payload structure.

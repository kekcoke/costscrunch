# Changelog — 2026-04-24

## CI/CD Infrastructure
- **Enhanced Integration Testing**: Modified `.github/workflows/workflow.yml` to support a local Cognito instance during integration tests.
    - Added `jagregory/cognito-local` container to the integration job.
    - Integrated `scripts/provision-local-cognito.js` for automated User Pool and Client creation.
    - Added health checks for both LocalStack and Cognito Local.
    - Implemented dynamic environment variable injection for Cognito IDs using `jq`.
- **Service Configuration**: Updated `infrastructure/docker-compose.localstack.yml` to include the `cognito` service in the `SERVICES` environment variable.

## Authentication & Build
- **Logout Functionality**:
    - Added "Sign Out" button to `App.tsx` and implemented `authApi.logout()` flow.
    - Registered `/auth/logout` route in Express local server and LocalStack bootstrap.
    - Verified global session invalidation via Cognito `AdminUserGlobalSignOutCommand`.
- **Bug Fixes**:
    - Resolved LocalStack build error where `AuthFunction` artifact was missing in `bundle-local.mjs`.

## Infrastructure Registrations
- **CDK Stack** (`infrastructure/stacks/CostsCrunchStack.ts`):
    - Added `POST /auth/logout` route (authenticated) using Cognito JWT authorizer.
    - Added `DELETE /auth/account` route (authenticated) for account deletion.
- **SAM Templates** (`template-x64.yaml` & `template-arm.yaml`):
    - Added `AuthFunction` with all auth routes: register, confirm, login, refresh, forgot-password, confirm-password, confirm-mfa, logout, delete account.
    - Added `UserPoolId` and `UserPoolClientId` parameters for Cognito integration.
- **Express Server** (`backend/src/server.ts`):
    - Added all missing auth routes: `/auth/register`, `/auth/confirm`, `/auth/login`, `/auth/refresh`, `/auth/forgot-password`, `/auth/confirm-password`, `/auth/confirm-mfa`, `/auth/logout`, `/auth/account`.
- **Frontend** (`frontend/src/App.tsx`):
    - Extracted inline logout handler to `handleLogout` callback for better code organization.

## Local Development Setup
- **Docker Compose** (`infrastructure/docker-compose.localstack.yml`):
    - Added `cognito-local` container (jagregory/cognito-local) on port 9229.
    - Added `localstack-cognito` one-shot container to auto-provision Cognito User Pool and test users.
    - Updated seed dependencies to wait for Cognito provisioning before seeding DynamoDB.
    - Set `MOCK_AUTH=false` to use real Cognito emulation instead of DynamoDB mock.
- **SAM env.json** (`infrastructure/sam/env.json`):
    - Added `COGNITO_ENDPOINT`, `USER_POOL_ID`, and `USER_POOL_CLIENT_ID` parameters.
    - Added `AuthFunction` and `ProfileFunction` environment blocks.
    - Set `MOCK_AUTH=false` for all functions.
- **Cognito Provisioning** (`scripts/provision-local-cognito.js`):
    - Enhanced with health check waiting and idempotent provisioning.
    - Added test user creation (test@costscrunch.dev, jane@costscrunch.dev, bob@costscrunch.dev).
    - Added user group assignment (pro, member).

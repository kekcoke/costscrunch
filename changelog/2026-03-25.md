# 2026-03-25

## Duplicate Receipt Detection

### Features
- **Deterministic hash computation** (`computeReceiptHash`) from normalized (merchant + date + amount)
- **Fuzzy matching** with Levenshtein distance (threshold: 10 edits) and amount tolerance (±$0.50)
- **ReceiptHashIndex GSI** for fast duplicate lookups in DynamoDB
- **EventBridge integration** emitting `DuplicateReceiptDetected` events

### Backend Changes

**`backend/src/utils/receiptHash.ts`**
- `computeEntityHash()` — SHA-256 hash from pipe-separated parts
- `normalizeReceiptFields()` — lowercase merchant, date-only (YYYY-MM-DD), amount rounded to 2 decimals
- `computeReceiptHash()` — convenience wrapper for receipt duplicate detection

**`backend/src/utils/fuzzyMatch.ts`**
- `levenshtein()` — O(n*m) single-row DP algorithm
- `fuzzyMatchReceipt()` — compares receipts with configurable tolerances
- Returns `FuzzyMatchResult` with `isDuplicate`, `similarity` (exact/fuzzy/none), `merchantDistance`, `amountDifference`

**`backend/src/lambdas/sns-webhook/index.ts`**
- Added `checkForDuplicate()` — queries GSI, uses fuzzy matching
- Added `emitDuplicateDetected()` — emits `DuplicateReceiptDetected` EventBridge event
- Writes `receiptHash`, `gsi3pk`, `gsi3sk` to expense records
- Graceful handling for missing GSI in dev/test environments

**`infrastructure/stacks/CostsCrunchStack.ts`**
- Added `ReceiptHashIndex` GSI (PK: `receiptHash`, projection: ALL)

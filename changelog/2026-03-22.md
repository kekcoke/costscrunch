# 2026-03-22 — LocalStack REST v1 Routing & CORS Hardening

## [Infra-P0] — API Gateway Resource Tree, Idempotent Bootstrap, Frontend Env Fix

### Added
- **Hierarchical Resource Tree**: `bootstrap.sh` now builds all 18 API Gateway resources explicitly (was flat — only top-level paths). Nested routes like `/groups/{id}/balances`, `/receipts/{expenseId}/scan`, and `/analytics/chart-data` are no longer 404-before-Lambda.
- **Scan Polling Endpoint**: `GET /receipts/{expenseId}/scan` in receipts Lambda — queries DynamoDB for Textract scan results by `RECEIPT#{expenseId}` / `SCAN#` prefix.
- **Dual Event Shape Detection**: `isApiGatewayEvent()` now detects both REST v1 (`httpMethod`) and HTTP v2 (`routeKey`) events. `resolveRoute()` normalizes both into a consistent `METHOD /path` string.

### Changed
- **Idempotent Bootstrap**: `bootstrap.sh` queries `get-rest-apis` first, creates only if no matching API exists. Previously created a new API (and ID) on every run, accumulating orphaned APIs.
- **Frontend Base URL**: `.env.dev` now requires the full REST API v1 invoke path (`/restapis/{ID}/{stage}/_user_request_`). Bare `http://localhost:4566` hit S3, not API Gateway.
- **Vite Config**: `loadEnv("test")` hardcoded mode replaced with `loadEnv(mode)` + `.env.dev` fallback. Shell env (from `dev:opt2` script) still takes priority.
- **Client Instantiation**: `receipts/index.ts` switched from raw `new S3Client({})` / `DynamoDBDocumentClient.from(new DynamoDBClient({}))` to factory functions `createS3Client()` / `createDynamoDBDocClient()` for consistent LocalStack endpoint configuration.
- **CORS Headers in Response Helpers**: `ok()` and `err()` in `receipts/index.ts` now include `Access-Control-Allow-Origin`, `Access-Control-Allow-Credentials`, and `Content-Type` headers (required for REST v1 proxy integrations — browser rejects headerless responses).
- **Route Matching**: receipts Lambda switched from exact `routeKey === "POST /receipts/upload-url"` to substring `route.includes("/upload-url")` to work with both REST v1 and HTTP v2 route formats.
- **Path Rename**: `/analytics/chartData` → `/analytics/chart-data` across backend handlers, tests, and API definitions.

### Fixed
- **Missing Closing Paren**: `receipts/index.ts` handler wrapped with `withLocalAuth(withErrorHandler(...))` but only had one closing `)` — caused esbuild build failure.
- **TypeScript TS2554 (94 errors)**: `withLocalAuth` required `context` parameter but all Lambda handlers and tests only pass `(event)`. Made `context` optional in input and output signatures.
- **Production URL Leak**: `vite.config.ts` `loadEnv("test")` loaded wrong `.env` file → `VITE_API_URL` was `undefined` → `??` fallback hit `https://api.costscrunch.io` during local dev.
- **4 Stale APIs Cleaned Up**: Deleted orphaned APIs (`2mhpmcfngl`, `9tseln10s7`, `59ki2y38fk`) accumulated from non-idempotent bootstrap.

SAM CLI local invoke uses HTTP API v2 event format — needs separate verification that existing `_local/` wrappers and the updated `isApiGatewayEvent` / `resolveRoute` logic work correctly under `sam local start-api`.
